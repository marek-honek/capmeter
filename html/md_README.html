AVR-tools

Tools for development of AVR microcontrollers.

Project wizard

First step

Open the terminal and go to a path where you want to create your AVR project. Use the cd command to change directory.

Getting the project template

Getting the template is easy. Just clone this repository.

1 # if you prefer SSH
2 git clone git@github.com:wykys/AVR-tools.git
3 # else use HTTPS
4 git clone https://github.com/wykys/AVR-tools.git

Set the name of the project folder

Use the mv command to change the folder name.

1 # replace the the_name_of_your_project with the desired name
2 mv AVR-tools the_name_of_your_project
3 # and go to your project folder
4 cd the_name_of_your_project

Open project in editor

Open your project in your favorite text editor. Choose one option.

1 # for Atom
2 atom .
3 # for Visual Studio Code
4 code .

Makefile settings

Makefile describes the build of a project. It also allows you to run auxiliary tools used for programming the target processor, analyzing compilation results and so on.

The Makefile file is located in the root directory of your project. Using a text editor, you can edit variables as required by the application.

Project name

It affects names of generated files.

1 # find the following line
2 TARGET = DEMO
3 # change DEMO for the name of your project
4 TARGET = the_name_of_your_project

Target microcontroller

The names of available microcontrollers are shown here.

1 # find the following line
2 CHIP = atmega328p
3 # change CHIP
4 CHIP = the_name_of_your_chip

Code size optimization

It affects size and speed of the application.

OPT DESCRIPTION
-O0 optimization is off
-O1 optimization level 1
-O2 optimization level 2
-O3 optimization level 3
-Os size optimization
1 # find the following line
2 OPT = -Os
3 # change OPT
4 OPT = required_optimization

Changing the programmer

List of supported avrdude programmers is here

1 # find the following line
2 AVRDUDE = avrdude -p $(shell $(WTR)) -c arduino -P $(shell $(WFS))
3 # change AVRDUDE
4 # port is required only for some programmers
5 AVRDUDE = avrdude -p $(shell $(WTR)) -c your_programmer -P programmer_port

Usage of make

Makefile allows compilation, programming and debugging of applications.

1 # compiling the project, equivalen to make
2 make all
3 # flash content
4 make flash
5 # compiling the project and programming the flash
6 make build_and_flash
7 # deletes the compilation outputs
8 make clean
9 # start avrdude terminal
10 make terminal
11 # EEPROM dump_eeprom
12 make dump_eeprom
13 # flash all (flash, EEPROM)
14 make flash_all
15 # chip testing
16 make chip_test

Source files *.c and *.S are located in the src/ folder. Header files *.h are located in the inc/ folder. The compilation outputs (binary files *.hex, *.elf, code disassembler [*.lss, *.lst], dependency files *.d, batch files *.o, map file *.map) are located in the folder. Project scripts are located in the hidden folder .scripts.

Change of frequency

The F_CPU [Hz] constant used for delay functions is defined in the settings.h file. The settings.h file must be plugged into all files in which you want to use the constant. You can also add additional constants that affect the behavior of the entire program.

#ifndef F_CPU
#define F_CPU 16000000UL // Hz
#endif
